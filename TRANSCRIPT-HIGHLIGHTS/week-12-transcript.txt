⏺ Budget Insight Generator — Plan

  Where it lives

  The Insights page (/insights) is already a placeholder. This is the natural
  home — no new navigation, no new pages. We just build it out.

  ---
  Do we need a backend API route?

  Yes, both frontend and backend changes are needed.

  - The Ollama API runs on http://localhost:11434. Calling it directly from the
  browser can run into CORS issues and is generally not clean.
  - A Next.js API route (app/api/insights/route.ts) acts as a thin proxy: the
  frontend sends budget data to it, it builds a prompt and calls Ollama, and
  returns the result.

  └──────────┴───────────────────────────┴───────────────────────────────────┘

  No other files need to change.

  ---
  What data gets sent to the AI

  We do not send raw transactions. Instead, we compute a small summary on the
  frontend and send that:

  - Total income
  - Total expenses
  - Current balance
  - Spending by category (e.g. Food: $320, Rent: $1200, ...)
  - Number of transactions
  - Date range (earliest → latest transaction date)

  This keeps the prompt short, token-efficient, and avoids sending unnecessary
  data.

  ---
  Flow

  [Insights page]
    1. Reads transactions from localStorage
    2. Computes summary (same math as Summary page)
    3. User clicks "Generate Insight"
    4. POST /api/insights with the summary object
          ↓
  [API route]
    5. Builds a short prompt
    6. POST http://localhost:11434/api/generate
    7. Returns the generated text
          ↓
  [Insights page]
    8. Displays the paragraph of insight text

  ---
  What the prompt looks like (rough idea)

  You are a helpful personal finance assistant. Here is a summary of the user's
  budget:

  - Income: $X
  - Expenses: $X
  - Balance: $X
  - Spending by category: Food $X, Rent $X, ...
  - Period: Jan 5 – Apr 18, 2026

  Give a short, friendly, 3–5 sentence summary of their financial health and one
   or two practical suggestions.

  Assumptions to confirm before coding

  2. Should the insight generate automatically on page load, or only when the 
  user clicks a button? (Button is safer — avoids surprise API calls)
  3. Should the response stream (text appears word by word) or appear all at 
  once? (All at once is simpler)
